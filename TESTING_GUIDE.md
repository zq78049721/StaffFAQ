# StaffFAQ 部署测试指南

## 快速测试流程

部署成功后，按照以下步骤测试应用：

### 步骤 1：访问应用

打开浏览器访问：https://stafffaq-demo.streamlit.app/

**预期结果**：
- ✅ 页面正常加载
- ✅ 看到聊天界面
- ✅ 没有错误提示
- ✅ 侧边栏显示正常

---

### 步骤 2：测试文档上传功能

**操作步骤**：
1. 在侧边栏找到 "上传人事文档" 区域
2. 点击 "选择文件" 按钮
3. 选择一个 `.txt` 格式的人事文档
4. 点击 "处理文档" 按钮

**预期结果**：
- ✅ 文件上传成功
- ✅ 显示处理进度
- ✅ 提示 "文档处理完成"
- ✅ 显示处理的文档数量和分块数

**测试建议**：
- 测试 1-2 个小文件（< 10 KB）
- 测试中文文档
- 查看日志确认没有错误

---

### 步骤 3：测试问答功能

**操作步骤**：
1. 在聊天输入框中输入问题
2. 例如："年假有多少天？"
3. 点击 "发送" 或按回车键

**预期结果**：
- ✅ 显示 "思考中..." 或加载动画
- ✅ 返回答案（5-10 秒内）
- ✅ 答案基于文档内容
- ✅ 没有报错信息

**测试问题示例**：
```
1. 年假政策是什么？
2. 考勤时间是怎么规定的？
3. 试用期有多长？
4. 请假流程是怎样的？
5. 加班有补贴吗？
```

---

### 步骤 4：测试多轮对话

**操作步骤**：
1. 问第一个问题："年假有多少天？"
2. 等待回答后，问第二个问题："那病假呢？"
3. 继续第三个问题："两种假有什么区别？"

**预期结果**：
- ✅ 上下文保持连贯
- ✅ 每次回答都基于之前的对话
- ✅ 不会丢失历史信息
- ✅ 回答准确且不重复

---

### 步骤 5：测试边界情况

**测试 1：无文档时提问**
- 不上传任何文档，直接提问
- **预期**：提示需要先上传文档

**测试 2：上传非 TXT 文件**
- 尝试上传 `.pdf` 或 `.docx` 文件
- **预期**：提示不支持的文件格式

**测试 3：空文档**
- 上传一个空白的 TXT 文件
- **预期**：提示文档为空或处理失败

**测试 4：超大文档**
- 上传一个超过 100 KB 的文件
- **预期**：处理时间较长但最终成功，或提示文件过大

---

### 步骤 6：检查日志（重要！）

**操作步骤**：
1. 在应用页面右下角点击 **"Manage app"**
2. 点击 **"Logs"** 标签
3. 查看最近的日志输出

**检查要点**：
- ✅ 没有 `ERROR` 级别的日志
- ✅ 看到 "LLM 客户端初始化成功"
- ✅ 看到 "文档处理完成"
- ✅ 没有 `TypeError` 或 `ImportError`

**正常日志示例**：
```
[INFO] LLM 客户端初始化成功 (provider: deepseek)
[INFO] 开始处理文档：hrData.txt
[INFO] 文档处理完成，共 5 个文档块
[INFO] 用户提问：年假有多少天？
[INFO] 检索到 3 个相关文档块
[INFO] LLM 响应生成完成
```

**错误日志示例**（需要修复）：
```
[ERROR] ModuleNotFoundError: No module named 'xxx'
[ERROR] TypeError: ...
[ERROR] KeyError: 'DEEPSEEK_API_KEY'
```

## 常见问题排查

### 问题 1：页面白屏或一直加载中

**可能原因**：
- 依赖包安装失败
- 应用启动超时

**解决方法**：
1. 查看 Logs 确认错误信息
2. 清除缓存（Manage app → Clear cache）
3. 重新部署（推送新代码）

### 问题 2：问答返回错误或超时

**可能原因**：
- API Key 未配置或错误
- 文档未处理
- 模型响应慢

**解决方法**：
1. 检查 Secrets 配置
2. 确认已上传文档
3. 查看 Logs 中的错误信息
4. 尝试简单问题测试

### 问题 3：文档上传失败

**可能原因**：
- 文件格式不支持
- 文件过大
- 编码问题

**解决方法**：
1. 确保是 `.txt` 格式
2. 文件大小 < 100 KB
3. 使用 UTF-8 编码保存
